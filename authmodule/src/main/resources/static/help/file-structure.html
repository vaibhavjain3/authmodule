File Structure

After downloading the Module, you will get a file structure as shown below. Types of file in the module are :

  • The src directory contains all of the source material for building the project, its site and so on. It contains a subdirectory for each type : main for the main build artifact, test for the unit test code and resources, site and so on.
  • The resources folder is dedicated to all the static resources, templates and property files.
    • resources/static - It contains static resources such as CSS, js, and images.
    • resources/templates - It contains server-side templates which are rendered by Spring.
    • resources/application.properties - It contains application-wide properties.
  • The pom. xml file contains information about the project and configuration information such as dependencies, build directory, source directory, test source directory, plugin, goals etc.
  • The target directory is used to house all output of the build.

The file tree below will describes more clearly about the file structure.

Important Files :

The following files can be configured according to the user's requirements :
File Name Information
application.properties The file contains database configuration and credentials for Oauth authorisation. Application properties support us to work in different environments
hibernate.cfg.xml Hibernate requires a set of configuration settings related to database and other related parameters. All such information is supplied as an XML file named hibernate.cfg.xml.
CustomUserDetails.java It is the Entity Class. The class data members and methods can be configured according to the fields in your database.
SpringSecurityConfig.java The configure() method of this class can be used to configure the access of html pages by the users with different roles.
Template/admin.html This page is restricted to the user with ADMIN role and can be designed accordingly.
Template/user.html This page can be accessed by any logged in user.
Template/login.html This is customizable login form for authentication.
Template/logout.html This is a customizable page to log out of the application.
Template/index.html This is the first page which opens after any user logs in. It can be customized as needed.
